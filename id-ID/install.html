Catatan instalasi Rust

Manajemen toolchain dengan rustup

Rust di instal dan dimanajemen oleh rustup. Dalam waktu 6 minggu dengan proses pelepasan yang cepat dan mendukung sejumlah platform yang besar , Jadi ada banyak pembuatan Rust yang tersedia kapan saja. rustup mengelola ini yang dibangun secara konsisten di setiap platform yang didukung Rust, memungkinkan instalasi Rust dari saluran rilis beta dan nightly serta dukungan untuk target cross-compilation tambahan

Untuk informasi lebih lanjut lihat dokumentasi rustup .

Konfigurasi PATH variabel lingkungan

Di lingkungan pengembangan Rust, semua tool diletakkan ke direktori ~/.cargo/bin %USERPROFILE%\.cargo\bin , dan ini adalah direktori di mana anda akan menemukan toolchain Rust, termasuk rustc, cargo, dan rustup.

Oleh karena itu, biasanya pengembang Rust menyertakan direktori ini di lingkungan variabel PATH mereka. Selama proses instalasi, rustup, akan mencoba untuk mengkonfigurasi PATH, tapi karena perbedaan antar platform, command shells, dan bugs di rustup, modifikasi untuk PATH mungkin tidak berlaku selama konsol belum di restart, atau pengguna logged out terlebih dahulu, atau mungkin tidak berhasil sama sekali.

Jika instalasi telah selesai, namun ketika menjalankan perintah rustc --version terjadi kegagalan, ini adalah alasan permasalahan yang biasanya terjadi.

Pertimbangan untuk Windows

Di Windows, Rust membutuhkan tambahan C++ build tools untuk Visual Studio 2013 atau setelahnya. Cara yang mudah untuk memperoleh build tools tersebut adalah dengan menginstal Microsoft Visual C++ Build Tools 2015 yang hanya menyediakan Visual C++ build tools. Sebagai alternatif, anda dapat menginstal Visual Studio 2015 atau Visual Studio 2013 dan selama proses instalasi pilih "C++ tools".

Untuk informasi lebih lanjut tentang konfigurasi Rust di Windows silakan lihat dokumentasi Windows-specific rustup.

Metode instalasi lainnya

Instalasi yang dijelaskan diatas, melalui rustup, adalah cara untuk menginstal Rust yang banyak disukai oleh pengguna, namun Rust juga dapat diinstal dengan cara yang lainnya.
